CSS Grid Basics

What is CSS Grid?

CSS Grid is a powerful layout system that allows you to create two-dimensional layouts with rows and columns. It's perfect for creating complex web layouts that were previously difficult to achieve with CSS.

💡 Key Concept

CSS Grid is a 2D layout system, unlike Flexbox which is primarily 1D. Grid allows you to control both rows and columns simultaneously.

Basic Grid Structure

To create a grid, you need a container element with display: grid and child elements that become grid items.

.container { display: grid; grid-template-columns: repeat(3, 1fr); gap: 10px; }

Grid vs Flexbox

Feature CSS Grid Flexbox
Layout Type 2D (rows & columns) 1D (row OR column)
Best For Page layouts, complex arrangements Component layouts, content distribution
Container Control High - controls both dimensions Medium - controls one dimension
Item Positioning Grid lines, areas Order, flex properties

CSS Grid Properties

Container Properties

display

Defines the element as a grid container.

.container { display: grid; /* Creates a block-level grid */ display: inline-grid; /* Creates an inline-level grid */ }

grid-template-columns

Defines the columns of the grid.

.container { grid-template-columns: 200px 200px 200px; /* Fixed widths */ grid-template-columns: 1fr 1fr 1fr; /* Fractional units */ grid-template-columns: repeat(3, 1fr); /* Repeat function */ grid-template-columns: minmax(200px, 1fr) 2fr 1fr; /* Minmax function */ }

grid-template-rows

Defines the rows of the grid.

.container { grid-template-rows: 100px 200px 100px; /* Fixed heights */ grid-template-rows: repeat(3, 1fr); /* Equal heights */ grid-template-rows: minmax(100px, auto) 1fr 100px; /* Mixed units */ }

gap

Defines the gap between grid items (shorthand for row-gap and column-gap).

.container { gap: 20px; /* Same gap for rows and columns */ gap: 20px 10px; /* row-gap column-gap */ row-gap: 20px; /* Gap between rows only */ column-gap: 10px; /* Gap between columns only */ }

Item Properties

grid-column & grid-row

Position items on specific grid lines.

.item { grid-column: 1 / 3; /* Start at line 1, end at line 3 */ grid-row: 2 / 4; /* Start at line 2, end at line 4 */ grid-column: span 2; /* Span 2 columns from current position */ }

justify-self & align-self

Align individual items within their grid area.

.item { justify-self: start | end | center | stretch; align-self: start | end | center | stretch; }

Grid Alignment

Container Alignment Properties

justify-items

Aligns all grid items along the inline (row) axis.

.container { justify-items: start | end | center | stretch; }

align-items

Aligns all grid items along the block (column) axis.

.container { align-items: start | end | center | stretch; }

justify-content & align-content

Aligns the entire grid content within the container.

.container { justify-content: start | end | center | stretch | space-around | space-between | space-evenly; align-content: start | end | center | stretch | space-around | space-between | space-evenly; }

Item Alignment Properties

justify-self & align-self

Align individual items within their grid area.

.item { justify-self: start | end | center | stretch; align-self: start | end | center | stretch; }

🎯 Pro Tip

Use place-items as a shorthand for align-items and justify-items. Use place-content for align-content and justify-content.

Grid Template Areas

What are Grid Areas?

Grid template areas allow you to create named grid areas and position items by referencing these names. This makes complex layouts much easier to understand and maintain.

Basic Grid Areas

.container { display: grid; grid-template-areas: "header header header" "sidebar main main" "footer footer footer"; grid-template-columns: 1fr 2fr 1fr; grid-template-rows: 80px 1fr 80px; } .header { grid-area: header; } .sidebar { grid-area: sidebar; } .main { grid-area: main; } .footer { grid-area: footer; }

Responsive Grid Areas

You can change grid areas for different screen sizes using media queries.

/* Desktop layout */ .container { grid-template-areas: "header header header" "sidebar main main" "footer footer footer"; } /* Mobile layout */ @media (max-width: 768px) { .container { grid-template-areas: "header" "main" "sidebar" "footer"; grid-template-columns: 1fr; } }

Empty Areas

Use dots (.) to create empty grid areas.

.container { grid-template-areas: "header header header" "sidebar main ." "footer footer footer"; }

2. Mobile-First Approach

Start with mobile layout and enhance for larger screens using progressive enhancement.

/* Start with mobile */ .container { display: grid; grid-template-columns: 1fr; gap: 20px; } /* Enhance for larger screens */ @media (min-width: 768px) { .container { grid-template-columns: repeat(2, 1fr); } } @media (min-width: 1024px) { .container { grid-template-columns: repeat(3, 1fr); } }

3. Use Grid Areas for Complex Layouts

Grid areas make complex layouts more readable and maintainable.

.layout { display: grid; grid-template-areas: "header header header" "nav main sidebar" "footer footer footer"; grid-template-columns: 200px 1fr 200px; grid-template-rows: 80px 1fr 80px; } .header { grid-area: header; } .nav { grid-area: nav; } .main { grid-area: main; } .sidebar { grid-area: sidebar; } .footer { grid-area: footer; }

4. Combine Grid with Flexbox

Use Grid for overall layout and Flexbox for component-level alignment.

/* Grid for layout */ .container { display: grid; grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); gap: 20px; } /* Flexbox for component alignment */ .card { display: flex; flex-direction: column; justify-content: space-between; }

5. Performance Considerations

  • Avoid deeply nested grids
  • Use auto-fit instead of auto-fill when you want items to expand
  • Limit the number of grid items for better performance
  • Use will-change: transform for animated grid items

6. Accessibility

  • Maintain logical tab order
  • Use proper heading hierarchy
  • Ensure sufficient color contrast
  • Test with screen readers

🚀 Pro Tips

  • Use grid-template shorthand for cleaner code
  • Leverage minmax() for flexible, responsive layouts
  • Use subgrid (when supported) for nested grid layouts
  • Consider using CSS Grid for form layouts
